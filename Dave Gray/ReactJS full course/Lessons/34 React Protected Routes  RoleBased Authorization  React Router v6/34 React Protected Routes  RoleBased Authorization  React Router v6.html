34 React Protected Routes RoleBased Authorization React Router v6 concept :

A] main concept , features or main components , used tools of creating [ Login Form] with protected Routs Submit :


Main Concept :

  • Combining the both previous [Login Form] & [Registration Form] with one app of several inner pages with providing protected routes autherization
  • [Login Form] should be contlored [Roled based] information inserted with several levels of Authorizations as next :
    • [Admin] => Sould be authorized to get access into all inner pages & [Admin page] , and NOT authorizierd to get access into [ Editor page ]
    • [Editor] => Sould be authorized to get access into all inner pages & [Editor page] , and NOT authorizierd to get access into [ Admin page ]
    • [user] => Sould be authorized to get access into all inner pages & [ page] , and NOT authorizierd to get access into [ Editor page ]
  • Creating a [Login Form] that using [Axios] api for creating the [post] process and submit user information for authetication
  • The [Login Form] will provide the Authentication by using both of the [context] & [real endpoint of node server - will be provided withni Node course -]
  • The [Login Form] will provide the Authentication by using both of the [context ] & [real endpoint of node server - will be provided withni Node course -]


    • Main Features or components of the [ login Form] :

    this form is about a Login Form with using the following frontend contents :

    1. 1- Conditional successful login message
    2. 2- Login Form :
      • Username input field
      • password input field

    3. 3- Sign-up section => incase of the user don't have an acount yet and need to create an account first


    Main Tools & Libraries requried for protecting the desired routers pages :

    This [Login Form] is about a Login Form with using the folowing tools :

    1. [Axios] library for asynchoronous responding process [post]
    2. {Context} : for define the [Authentication Provdier function] which will include the main Authentication coinfguration settings to store the main inserted data of the [Login Form]
    3. [node.js server] + [Mogo DB] connection -> [available in the Node course ] to provide the main endpoints of server's [response data] & [response accessToken] & [response roles]



    B] main steps syntax of protecting the desired routers of pages app :


    the syntax , as following :

    • 1] [ hooks/useAuth.js ] : Creating a custom hook of {useAuth} for reducing the required steps of uaing defined contex inside each protected route :
      1. This save will save both steps of [importing {useContext}] + [import the context variable [AuthContext] ] , and will be have to import only the { Define the custom hook} ,to be able to remove the following form inside each route file :
        • Imported { useContext} hook
        • Imported {AuthContext} varaible file

        and instead of preious (2) steps will just imprt the custom hook defined function {useAuth} from [ useAuth.js ] file inside this route file


    • 2] Modifying defined Routes inside {app.js} => with using protector file {RequiredAuth.js} & adding defined Roles that will protect our pages routes , by using the next procedures :

      to use different protector route with different role , file as the next code :
      define the protected route {Home} : Authorized for public {User} by assigning code of {2001} :
      < Route element={ < RequriAuth > } allowedRoles={[ 2001 ]} >
      < Route path='/' element={ < Home > } />
      < /Route >
      define the protected route {Home} : Authorized for public { Editor } by assigning code of {1985} : :
      < Route element={ < RequriAuth > } allowedRoles={[ 1985 ]} >
      < Route path='/admin' element={ < Admin > } />
      < /Route >
      define the protected route {Home} : Authorized for public { Admin } by assigning code of {5150} : :
      < Route element={ < RequriAuth > } allowedRoles={[5150]} >
      < Route path='/editor' element={ < Editor > } />
      < /Route >
      define the protected route {Home} : Authorized for public { Admin & Editor } by assigning both codes of {5150} & {1985} : :
      < Route element={ < RequriAuth > } allowedRoles={[ 1985, 5150 ]} >
      < Route path='/lounge' element={ < Lounge > } />
      < /Route >

    • 3] Creating {components/RequriAuth.js} => the protector file that will protect our pages routes , by using the next procedures :

      // A] Importing section :

      • 1- import the defind [custom hook] function => to be use the short method of custom hook - when ned to use the inner prop inside the context - :
        import useAuth from '../hooks/useAuth' ;

      • 2- import several classes from the [react-router-dom] :
        import { useLocation, Navigate, Outlet } from 'react-router-dom';


      // B] Define the main function for fetching the authorization level + protecting the Routes + with using {allowedRoles} parameter to hold the role code value by calling it inside the defined route wihtin the [App.js] :
      const RequireAuth = ({ allowedRoles }) => {
      // 1- Extracting the [auth] prop that defined inside the main context provider function , [by using the imported custom hook] :
      const {auth} = useAuth() ;
      // 2- Define a extracted location variable [ the current location ] : :
      const location = useLocation() ;

      /* 3- return the main { RequireAuth} functional component of [ checking for roles] according to the next nested conditons :
      1- if the [ passed parameter {allowedRoles} include a defined Role => then navigate to the protected children wraped routes inside the protector {RequireAuth} route ] by return the {Outlet}
      2- if the [ passed parameter {allowedRoles} NOT include a defined Role => Check for there is a user =>
      a- if there is a user => navigate to the {UnAuthorized} page
      b- if there is NOT a user => navigate to the {Login} page : */

      return (
      auth?.roles?.find(role => allowedRoles?.include(role)) ?
      < Outlet / >
      : auth?.user ?
      // [No Role + No user ]
      < Navigate to='/unauthorized' state={ { from: location } } replace / >
      // [No Role + but there is a user ] => adding the {state} attribute with locaton will allow the browser to remember where the user were
      : < Navigate to='/Login' state={ { from: location } } replace / >
      )
      }


    C] Vip notes about protected role based routes Form :

      1. Must Wrap the protected route(s) within the defined {protecting component} route
      2. The defined {protecting component} route > can include more that one protected route with the same code
      3. Inside the protector route must belonging role parameter [ as well it defined inside the {protector functional compoennt } ] with assined code value
    • incase of defne and use a custom hook function to reduce the required steps of using deifned context, it must be imported inside the [{protecting component} file] where it will be used in fetching the isnerted infromation of the Login Form
