  2. 2] Properties Differences betweeen the [MongoDB] as a NoSQL & the [MySQL] as traditional database :
    1. [ MySQL] Database :

      It is a traditional database based on a [Relational structure] that contians relational refernces Tables that refernces each other using the [join] as Data Queries storing

      [Relational Tables] in sql database is normalizing the data , which is called [DRY : Do not repeate yourself] , that means the data is NOT duplicated


    2. [MongoDB] (NoSQL) Database :

      It is based on the [Collection] & [documents ] [NO Relational structure]

      It is storing the data using [collections] , each collection contains [documents] as end visual records and rows

      Each [document] => contains data stored as [json format] , that has a [key value] structure

      [ collection data ] example that holds all data about a user (instead of a relational table ) :
      // [User collection] :
      {
      _id : ObjectId("66648799") , // [ this is instead fo using the table key ]
      username: "value" ,
      password : "$sdkf" ,
      roles : { User : 2001 , Editor : 1984 } ,
      refreshToken : "sdf$sdfd6"
      }



  3. 3] Main advantages of using the [MongoDB] database :

    a- [Perfomance] :

    • Very speedy in of the [Collection] of collecting the data from it's [Documents ]

    b- [ Flexibility ] :

    • very easy to modify the structure data as :

      create a new data sturuture without need to very complex of modification the main data , which is more simmiler to add a new property into an object

    c- [ Scalablity ] :

    No such a SQL databse can create a larger data structure with high speed request responding with low rate of lateness

    d- [ Usability ] :

    No such other database can be set up much easer by using the [mongoDB] cloud very fast {mongodb.com}
