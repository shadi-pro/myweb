A] The main concept of [Roled based Authorization] :


1] The main difference between the [Authentication] and [Authorization] :

  1. 1] [Authentication] concept :

    It is about [process of verfying who user/someone is]

    It is the first stop of dealing with backend => [process of login by the username and password] by using [JWT tool] including both of [access token] and [refresh token]



  2. 2] [Authurization] concept :

    It is about [process of what user's resources has access to]

    It is the second step after the [login user confirm Authentication] process => [process of what user's resources has asccess to] by using also [JWT tool]

    This processs will use the JWT and it's Token to allow access to user API Endpoints which will provide user's resources data

    This processs will use [Authorization Header]


2] The main concept of [User Roles and permissions ] :

  1. This lesson will expand the [Athorization] process by adding {[user roles] with spcecific permissions} to the Api Authorization process
  2. [User Roles] are different levels of access to the endpoint resources data for each level
  3. [User Roles] are being sent to the access token payload , and verifed by middleware
