[Middleware] main concept : is any in between {request} and {response} handler in the web framework for [node.js] , so {previous method of the {app.get} of route handler is considering a type of [middelware]

C] Notes about building [Node web server] using , [express.js] Framework :

  • The main offfical website of the express.js framework is : { expressjs.com }
  • this lesson is bulding the node web server with using only the node JS syntax and envoirnment (witht using/utilizing Framework of :express.js] )
  • {app.get('request url - accept a regular expressions - ', ( req , res, next ) {})} => used to define a certain route , and than adding the certain hanlding procedures will ber implementted awhen this reqest url is being called , which can include folwlowing procdueres :
    1. {res.send('textual value') } => which will sent this value in the current page when the assinged request url is being called
    2. {res.send(File() } => which will send a certain file to be displayed when the assinged request url is being called
    3. {res.send(File() } => has several syntaxes as following :
      1- the Basic method of the node =>
      { res.sendFile( './views/FileName.extension' , {root: __dirname } ) }

      2- the path.join method =>
      { res.sendFile( path.join( __dirname ,'views' , 'FileName.extension' ) ) }
    4. Regular expression could be included inside the requested url used for controled url request , and can contain the folllwing :
      • '^' => means Must begins with the next assigned text , such as the next example :
        app.get('^/', (res , req) => {} ) ; => [the req route url must begins with the '/' ]

      • '$' => means Must Ends with the previous assigned text, such as the next example :
        app.get('/$', (res , req) => {} ) ; => [the req route url must ends with the '/' ]

      • '|' => means could include the previous assigned text or the next assinged text , such as the next example :
        app.get('/|index.html' , (res , req) => {} ) ; => [the req route url could be '/' OR 'index' ]

      • '(text)?' => means the in-between brackets is an optionally to be added , such as the next example :
        app.get('index(.html)?' , (res , req) => {} ) ; => [the req route url is 'index' and could include '.html' after the index ]

      • '*' => means to accepting any value will be come , such as the next example :
        app.get('/*' , (res , req) => {} ) ; => [the req route url is '/' and followed by any value ]

      • using Multiple regular expressions in one method => such as the next example :
        app.get('^/$|/index(.html)?' , (res , req) => {} ) ; => [the req route url Must start and ends with '/' OR '/index and could be followed by '.html' ]

      • The route defintion could include chained callback functions and also couuld include using of the [next()] partater to accesss to the next chained function , and this inlcuding (2) types of methods , as folowting :
        • res.get('', (res , req , next ) => {
          console.log ('hello ') ;
          next() ; // using the this parameter method will allow to move to the next chained function
          } , (res ,req) => {
          res.sendFile(path.join(full path of a file ) ) ;
          } ) ;
